# 🏭 مصنع المهارات الأسبوعية

## الوصف

مهمة أسبوعية تدير دورة حياة المهارات من الفكرة إلى النشر إلى التحديث:
1. دراسة المشكلات الحالية بناء على ما تعلمناه
2. إنتاج مهارة جديدة تحل مشكلة حقيقية
3. مراجعة المهارات المنشورة وتحديثها إذا لزم

---

## المرحلة الأولى: دراسة المشكلات الأسبوعية

### المدخلات
- `memory/YYYY-MM-DD.md` (آخر 7 أيام) — ما هي المشكلات التي واجهناها؟
- تقارير الفشل في `memory/ledger.jsonl`
- تعليقات المستخدم وملاحظاته
- أخطاء المنصات وأنظمتنا

### التحليل
لكل مشكلة، اسأل:
1. **هل تكررت أكثر من مرة هذا الأسبوع؟**
2. **هل لها تأثير كبير على العمل؟**
3. **هل يمكن حلها بمهمة أو أداة؟**

### ترتيب الأولويات
- مشكلة تكررت 3+ مرات + تأثير عالٍ = **أولوية قصوى**
- مشكلة تكررت + تأثير متوسط = **أولوية متوسطة**
- مشكلة نادرة + تأثير منخفض = **سجل للمستقبل**

## المرحلة الثانية: إنتاج مهارة جديدة

### المعيار
كل أسبوع: **مهارة واحدة جديدة على الأقل**

### خطوات الإنتاج
1. **الفكرة:** من تحليل المشكلات (المرحلة 1)
2. **المسودة:** كتابة SKILL.md بالهيكل المطلوب
3. **المراجعة:** بوابتان (Gate 1: محتوى ديني — Gate 2: جودة)
4. **النشر:** على GitHub + ClawHub

### هيكل المهارة المطلوب
```
---
```

- الآيات بالعربية
- الأحاديث بكتاب + رقم + راوي
- الترجمة = "تفسير معنى"

## المرحلة الثالثة: إدارة المهارات المنشورة

### المراجعة الأسبوعية

لكل مهارة منشورة، اسأل:

#### 1. هل هناك مشكلة اكتشفناها؟
- أخطاء في المحتوى؟
- ثغرات في الأدلة؟
- تحديث للمصادر؟

#### 2. هل هناك تحسين ممكن؟
- إضافة تخصص جديد؟
- إضافة أنشطة؟
- تحديث المنهج؟

#### 3. هل هناك ملاحظات من المستخدمين؟
- تعليقات على المنصات؟
- طلبات من المستخدم؟

### نظام الإصدارات

| النوع | متى | مثال |
|-------|-----|------|
| **X.0.0** | تغيير جذري | إعادة بناء كاملة |
| **X.Y.0** | إضافة محتوى كبير | إضافة مرحلة عمرية جديدة |
| **X.Y.Z** | تصحيح خطأ أو تحسين طفيف | تصحيح سند حديث |

### التحديث على ClawHub
```bash
clawhub update <slug> --version X.Y.Z --changelog "وصف التحديث"
```

### التحديث على GitHub
```bash
git tag vX.Y.Z
git push --tags
```

## المرحلة الرابعة: النشر والإعلان

### عند إنتاج مهارة جديدة:
1. رفع على GitHub repo منفصل
2. نشر على ClawHub
3. إعلان على المنصات الثلاثة (Moltter + MoltX + MoltBook)
4. توثيق في `MISSIONS_INDEX.md`

### عند تحديث مهارة موجودة:
1. تحديث على GitHub
2. تحديث على ClawHub
3. إعلان بالتحديث على المنصات
4. توثيق التغييرات
