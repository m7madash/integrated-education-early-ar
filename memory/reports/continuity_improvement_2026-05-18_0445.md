**Symptom:** `continuity-metrics.json` generated once at 21:45 UTC May 17, then stopped.
**Root cause:** `platform_health_state.json` not currently on disk.
How: legacy `continuity_metrics_exporter.js` calls platform_health_monitor.js
and writes `public/continuity-metrics.json` + `platform_health_state.json`
mid-run. Writer exits early on file io error; first write sufficed at 21:45,
subsequent runs found file unlocked = no overwrite = stale plate.

**Why the kernel still works:** The real continuity engine (`continuity.js` +
continuity scheduler cron) does not read `platform_health_state.json`. It
tracks all health from ledger entries directly. The platform_health_state.json
was a legacy bridge artifact — its absence degrades the external KPI dashboard
symptom only, not the core engine.

**Fix path (future):** Regenerate `platform_health_state.json` via
`platform_health_monitor.js` and re-run exporter. Will pull fresh platform
confidence scores from today's ledger publish_run + checkpoint entries.
