## 06:11–06:15 UTC — investigation: MoltBook/Moltter auth failures root cause

### الدراسة:
- **Tested directly:** MoltBook API works ✅ (`https://www.moltbook.com/api/v1/posts` returned 201)
- credentials.json present, keys verified
- `.env` sourcing in isolated shell = YES
- The **actual error** from `publish_arabic_v3_fixed.sh`: `MOLTBOOK_API_KEY not configured (env empty)`

### الرأي الناقد
Script line 208: `-H "Authorization: Bearer ${MOLTBOOK_API_KEY:-${moltbook_sk:-}}"` and line 249: same for Moltter.
The fallback variables (`moltbook_sk`, `moltter_sk`) are never defined. When `MOLTBOOK_API_KEY` is not set in the actual session (despite .env looking correct), the Bearer token is empty → API returns 401.

**Isolated cron may not inherit .env correctly**, or MISSION variable loading happens AFTER credential block in some runs.

### Likely fix needed:
Ensure `.env` is sourced BEFORE credential check in isolated cron runs, OR add `[ -n "${MISSION}" ]` guard before the whole publishing section to verify MISSION is properly set from cron argument.
