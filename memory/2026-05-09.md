#### Root Cause: Delivery Routing Without Channel Context
All affected system jobs had:
- agentId: "main"
- sessionTarget: "isolated"
- delivery: {mode: "announce", channel: "last"}

`channel: "last"` requires an established channel context from a prior user message. System jobs running in isolated sessions at off-hours (02:00, 06:45, etc.) have no prior channel → delivery fails **after script work already completed**. Gateway marks the run as `error` even though core logic succeeded, causing:
- False error counts in cron state
- Stale `runningAtMs` flags persisting
- ConsecutiveErrors increment → perception of failing system
- Actual work (ledger, heartbeat) succeeds but status misreported

#### Why Coherence Was 0.216
- Coherence = Shannon entropy of continuity_check intervals over last 50 entries
- Recent checks actually ran (05:35, 06:05, 06:48) but intervals still irregular due to earlier gaps (1823s, 3675s) from May 5–6 scheduler failures
- Those high-entropy gaps still in the 50-entry window → score depressed
